# 配置管理模块 — 开发设计计划

## 一、背景与目标

参考 SolarWinds NCM（Network Configuration Manager）等产品的**网络配置（Network Configuration）**能力，在 NetOps 中新增独立的**配置管理模块**，用于：

- 对网络设备配置进行**集中备份、版本管理、变更跟踪**；
- 提供**配置摘要、合规检查、变更模板**等能力；
- 与现有「设备连接」「配置模板」「流程编排」打通，形成完整的配置生命周期管理。

**界面语言**：功能截图参考为英文，本系统**界面与菜单统一使用中文**。

---

## 二、参考能力与中文菜单结构

参考 SolarWinds 配置管理类模块的典型结构，本模块建议的一级菜单（左侧导航）如下，**全部为中文**：

| 英文（参考）           | 中文（本系统）     | 说明 |
|------------------------|--------------------|------|
| Config Summary         | **配置摘要**       | 仪表盘：设备数量、最近备份数、变更次数、合规状态概览等。 |
| Configuration Management | **配置管理**     | 设备配置的备份存储、版本历史、对比、回滚、检索。 |
| Config Change Templates | **配置变更模板**  | 可复用的变更模板（可与现有「配置管理组件」模板库打通或独立）。 |
| Compliance             | **合规**           | 合规策略、基线对比、合规报告。 |
| End of Support         | **服务终止**       | 设备/EOS 信息、生命周期提醒（可选）。 |

**说明**：作业（配置备份/下发/合规等）不在本模块实现，由**原子功能组件与流程编排**（工作流）中实现。

---

## 三、与现有系统的关系

| 现有能力 | 与本模块关系 |
|----------|----------------|
| **原子组件 - 设备连接** | 配置备份/下发时使用 SSH 连接配置与连接池。 |
| **原子组件 - 配置管理** | 当前为「下发用模板」；本模块「配置管理」侧重**设备侧拉取的配置备份**的存储与版本；「配置变更模板」可与现有模板库关联或复用。 |
| **原子组件 - 配置生成** | 生成的内容可进入「配置变更模板」或直接用于下发。 |
| **流程设计器 - 配置备份节点** | 需落地：执行时从设备拉取配置并写入本模块的配置库。 |
| **任务作业管理 / 流程编排** | 配置备份、下发、合规检查等作业在原子工作流中编排与执行，本模块不包含作业调度功能。 |
| **CMDB** | 设备列表、设备类型可来自 CMDB 或设备分组，用于按设备筛选配置。 |

## 四、功能子模块设计（中文）

### 4.1 配置摘要

- **入口名称**：配置摘要（中文）
- **能力**：统计卡片 + 简单图表
  - 已纳入配置管理的设备数、最近 24h/7d 备份成功数、失败数；
  - 最近变更次数、合规通过率；
  - 可选：最近备份列表、最近变更列表。
- **数据来源**：配置管理库的备份记录、流程执行产生的备份/变更记录、合规结果。

### 4.2 配置管理

- **入口名称**：配置管理（中文）
- **能力**：
  - **设备配置库**：按设备（或设备组）存储每次备份的配置内容（全文 + 元数据：设备、时间、来源、备注）。
  - **版本历史**：单设备的版本列表，支持查看、对比（diff）、回滚（可选）。
  - **检索**：按设备名/IP、时间范围、关键词检索备份记录或配置内容。
- **与现有「配置管理组件」的区分**：现有组件管「模板」；本模块管「从设备拉取并保存的配置备份」。

### 4.3 配置变更模板

- **入口名称**：配置变更模板（中文）
- **能力**：
  - 可复用变更片段或模板（如：标准 ACL、NTP、SNMP 等），支持与现有 config/files 模板库关联；
  - 模板可绑定设备类型、用途标签；
  - 变更前/后可与「配置管理」中的备份做对比或预检查。
- **实现方式**：可复用现有配置模板表，或单独建「变更模板」表并做类型/用途区分。

### 4.4 合规

- **入口名称**：合规（中文）
- **能力**：
  - **策略/基线**：定义合规规则（如：禁止某命令、必须包含某段配置、正则/键值检查）。
  - **执行与报告**：对指定设备或设备组执行合规检查，结果存为报告；支持按设备、时间查看历史合规结果。
- **可选**：与「配置管理」中某版本做基线对比。

### 4.5 作业（不在本模块实现）

- **说明**：配置备份、配置下发、合规检查等作业**不在本模块实现**，由**原子功能组件与流程编排（工作流）**中实现；本模块仅提供配置存储、版本、合规策略与报告等能力，供工作流调用或展示。

### 4.6 服务终止（可选）

- **入口名称**：服务终止（中文）
- **能力**：维护设备或型号的 EOS/EOL 信息，列表展示即将/已 EOS 的设备，提醒升级或替换。数据可来自手工维护或与 CMDB 关联。

## 五、菜单与路由建议（中文）

建议在 NetOps 左侧菜单中新增一级菜单，例如：

- **菜单名称**：**配置管理模块**（中文）
- **路由前缀**：`/config-module` 或 `/rpa/config-module`
- **子菜单（全部中文）**：

| 菜单项           | 路由建议                  |
|------------------|---------------------------|
| 配置摘要         | `/config-module/summary` |
| 配置管理         | `/config-module/management` |
| 配置变更模板     | `/config-module/change-templates` |
| 合规             | `/config-module/compliance` |
| 服务终止         | `/config-module/eos`（可选） |

## 六、数据与后端建议

- **设备配置备份库**：新建表存储每次备份（设备标识、备份时间、配置全文、元数据、可选版本号）；可考虑按设备分表或大字段+索引。
- **合规策略与结果**：策略表、执行结果表、报告表。
- **配置变更模板**：可复用现有 config 模板表并扩展类型/用途，或单独建表。
- **作业**：不在本模块实现，由原子工作流/流程编排与任务作业管理负责。

---

## 七、实施阶段建议

| 阶段 | 内容 | 说明 |
|------|------|------|
| 1 | **配置管理（备份库）** | 实现设备配置备份的存储、版本列表、查看与检索；流程中「配置备份」节点落地并写入此库（作业由原子工作流实现）。 |
| 2 | **配置摘要** | 基于备份记录与合规结果做统计与简单图表，配置摘要页。 |
| 3 | **配置变更模板** | 变更模板的维护与在流程中的选用。 |
| 4 | **合规** | 合规策略、执行与报告。 |
| 5 | **服务终止**（可选） | EOS 信息维护与展示。 |

---

## 八、总结

- 本计划参考 SolarWinds 类产品的配置管理能力，在 NetOps 中新增**配置管理模块**一级菜单，**界面与菜单统一使用中文**。
- 子功能包括：**配置摘要、配置管理、配置变更模板、合规、服务终止**；**作业**（配置备份/下发/合规等）不在本模块实现，由原子工作流（流程编排）实现。
- 实施时建议先做**配置管理（备份库）**与**配置备份节点落地**，再做摘要，最后做合规与变更模板、服务终止，以尽快支撑「网络设备配置备份」的完整闭环。
