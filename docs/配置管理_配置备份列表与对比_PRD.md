# 配置管理 — 配置备份列表与对比 产品需求文档（PRD）

**文档版本**：1.0  
**关联**：配置管理模块（配置摘要、配置管理、变更模板、合规、服务终止）中的「配置管理」页（备份列表与版本对比）。  
**依据**：在《配置管理模块_代码审查与改进建议》讨论基础上，对配置管理页的展示与交互做产品级约定。

---

## 1. 背景与目标

### 1.1 背景

- 配置管理页当前为「备份记录」维度的平铺列表（每条备份一行），展示设备标识、设备名、主机、来源、备份时间等；支持查看单条内容、与上一版对比、删除。
- 希望改为以**设备**为核心：同一设备只占一行，点击后查看该设备下的备份版本；列表不展示设备标识，仅展示设备名称与 IP；与作业关联的 ID 以作业侧为准；版本对比结果需高亮差异行。

### 1.2 产品目标

1. **以设备为入口**：一级列表按设备聚合（同 IP + 主机名视为同一设备），每设备一行；点击进入该设备的备份详情（版本列表）。
2. **展示与 ID**：列表不展示「设备标识」；展示「设备名称」「IP 地址」；业务上的 ID 与作业关联（见 2.1）。
3. **设备名称与 CMDB**：设备名称、IP 等展示优先从 CMDB 查询补齐，便于与资产一致。
4. **单设备备份上限**：同一设备最多保留 90 条备份；支持删除、下载、任意两版对比。
5. **配置比较体验**：版本对比页对差异行做高亮（如绿色底），便于阅读。

---

## 2. 功能需求

### 2.1 ID 与作业关联

| 需求 ID | 描述 | 验收标准 |
|---------|------|----------|
| F1.1 | 业务侧用于展示、追溯的「ID」以作业维度为准，而非配置备份表主键 | 列表/详情中展示的与执行相关的 ID 为**作业执行 ID**（或作业 ID），由写入备份时传入并落库；若当前写入未带 execution_id，需在备份表增加 `job_execution_id`（或 `job_id`）字段，流程/作业执行写入时传入 |
| F1.2 | 前端不将「备份表主键 id」作为主要业务 ID 展示给用户 | 设备维度列表可不展示备份 id；设备详情内版本列表可展示「执行 ID」或「备份时间」等，不强调内部 id |

**说明**：若备份来源于作业执行，建议备份表增加 `job_execution_id`（或 `job_id`），写入时由调用方传入，列表/详情展示该字段。

---

### 2.2 不显示设备标识，仅显示设备名称与 IP

| 需求 ID | 描述 | 验收标准 |
|---------|------|----------|
| F2.1 | 配置管理一级列表（设备列表）不展示「设备标识」(device_id) 列 | 表头与单元格均无设备标识列 |
| F2.2 | 配置管理一级列表展示「设备名称」「IP 地址」（主机）两列 | 设备名称、IP 地址（或主机名）为列表主要展示字段；若暂无名称则以 IP 或「-」占位 |

---

### 2.3 设备名称与 CMDB 关联展示

| 需求 ID | 描述 | 验收标准 |
|---------|------|----------|
| F3.1 | 设备名称、IP 的展示优先从 CMDB 查询 | 前端或后端在展示前，根据 device_host / device_id 查询 CMDB 资产，取「名称」「IP/主机名」等用于展示；若 CMDB 无则用备份表内 device_name、device_host |
| F3.2 | 主机列展示 IP（或主机名），设备名列展示设备名称（CMDB 名称或备份表 device_name） | 当前仅主机列有 IP 的问题消除；设备名列有明确展示且尽量来自 CMDB |

**实现建议**：后端提供「按 device_host 或 device_id 批量解析 CMDB 名称」的接口，或前端在渲染设备列表时按设备维度请求 CMDB 详情；若 CMDB 未对接可先仅用备份表字段。

---

### 2.4 同一设备在前端显示为一条，点击查看备份情况

| 需求 ID | 描述 | 验收标准 |
|---------|------|----------|
| F4.1 | **同一设备**定义为：device_host（IP/主机名）相同；若存在 device_id 且业务约定「同 device_id 即同设备」，则同 device_id 也可聚合 | 聚合键二选一或组合：仅 device_host；或 (device_id, device_host) 一致 |
| F4.2 | 一级列表为「设备维度」：每个设备一行，可展示设备名称、IP、该设备备份数量、最近备份时间等 | 列表数据来源于按设备聚合后的结果（后端聚合或前端聚合）；每行对应一个设备 |
| F4.3 | 用户点击某设备行后，进入该设备的**备份详情**（二级）：展示该设备下所有备份版本列表（时间倒序） | 支持从设备列表钻取到设备备份详情页或抽屉；详情内为版本列表，支持删除、下载、选择两版对比 |

---

### 2.5 同一设备备份最多 90 条；删除、下载、任意两版对比

| 需求 ID | 描述 | 验收标准 |
|---------|------|----------|
| F5.1 | 同一设备（同上聚合规则）下最多保留 **90 条**备份记录；超过时采用**滚动覆盖**，不报错 | 写入新备份时：若该设备已有 90 条，则**先删除该设备下创建时间最早的一条，再插入新备份**（先进先出）；接口正常返回 201，不返回错误。这样作业/流程可持续执行，无需用户先手动删除 |
| F5.2 | 用户可在设备备份详情内对单条备份执行**删除** | 调用现有 DELETE /backups/{id} 或等价接口；删除后列表刷新，且仍满足最多 90 条 |
| F5.3 | 用户可在设备备份详情内对单条或选中的备份执行**下载** | 下载当前版本配置为文本文件（如 .txt）；可选：多选后打包为 ZIP 下载 |
| F5.4 | 用户可在设备备份详情内**选择任意两个版本**进行配置比较 | 提供版本 A、版本 B 选择（下拉或勾选），确认后进入对比页/弹窗，展示 diff 结果 |

---

### 2.6 配置比较页差异行高亮

| 需求 ID | 描述 | 验收标准 |
|---------|------|----------|
| F6.1 | 配置比较页面（两版本 diff 结果）中，对**有差异的代码行**做高亮展示 | 差异行使用明显背景色（如**绿色底**）标记，与无差异行区分；新增/删除行若分开展示，也可用不同色（如绿/红）区分 |
| F6.2 | 比较结果仍支持滚动查看全文 | 高亮不改变原有滚动与排版，仅增加行背景样式 |

**实现建议**：后端 diff 接口可返回「行级 diff」结构（如每行带 add/remove/unchanged 标记），前端按行渲染并加样式；或前端基于统一 diff 文本解析出行类型后高亮。

## 3. 数据与接口

### 3.1 数据模型变更建议

| 表/字段 | 变更 | 说明 |
|---------|------|------|
| config_module_backups | 新增 `job_execution_id`（或 `job_id`）可空 | 写入时由作业执行流程传入，用于列表/详情展示「作业执行 ID」 |
| 同设备备份条数上限 | 业务规则：按 device_id（或 device_host）聚合后最多 90 条；超过时**滚动覆盖** | 写入时若该设备已有 ≥90 条，先删该设备下 created_at 最早的一条，再插入；不报错 |

### 3.2 接口建议

| 接口 | 变更/新增 | 说明 |
|------|-----------|------|
| GET /config-module/backups | 可选：支持「按设备聚合」模式 | 如 query 增加 `group_by_device=1`，返回按设备聚合的列表（每设备一条，含 backup_count、latest_created_at 等）；或保留现有列表，由前端按 device_host 聚合 |
| GET /config-module/backups/device/{device_id}/history | 保持，limit 建议最大 90 | 设备详情页用该接口拉取该设备下所有版本；后端可强制 limit≤90 |
| POST /config-module/backups | body 增加 `job_execution_id` 或 `job_id` 可选 | 流程/作业调用时传入，便于展示「执行 ID」 |
| 写入时超 90 条 | **滚动覆盖**：插入前检查该设备备份数，若 ≥90 则删除该设备下 created_at 最早的一条，再插入新记录；接口始终返回 201，不报错 | 不采用「报错并拒绝写入」策略；若后续产品希望改为报错，可再调整 |
| GET /config-module/backups/diff | 可选：返回行级 diff 结构 | 便于前端做行级高亮；若保持仅返回 diff_text，则前端需解析 unified_diff 后高亮 |

### 3.3 CMDB 对接

- 提供「根据 device_host 或 device_id 解析设备名称」的接口（或复用现有 CMDB 资产查询），供配置管理页展示设备名称、IP。

## 4. 前端规格

### 4.1 配置管理页结构（建议）

| 区域 | 内容 |
|------|------|
| 一级：设备列表 | 表格列：**设备名称**（优先 CMDB）、**IP 地址**（主机）、备份数量、最近备份时间；不展示设备标识；行可点击 |
| 钻取：设备备份详情 | 抽屉或二级页：该设备下备份版本列表（时间倒序）；每行支持「查看」「删除」「下载」；支持勾选两版后「对比」；列表最多展示 90 条 |
| 配置比较 | 独立页或大弹窗：左右/上下或统一 diff 文本展示；**差异行绿色底**（或增/删不同色） |

### 4.2 展示与交互要点

- 列表与详情中与「执行」相关的 ID 展示为作业执行 ID（或作业 ID），不强调备份表主键。
- 设备名称、IP 优先从 CMDB 获取；无则用备份表 device_name、device_host。
- 同一设备（IP + 主机名相同）在一级列表仅一条；点击后仅展示该设备下的备份。
- 单设备备份上限 90 条；删除、下载、任意两版对比均在设备备份详情内完成。
- 配置比较页对差异行做高亮（如绿色底）。

---

## 5. 验收标准汇总

- [ ] 配置管理一级列表不展示设备标识，展示设备名称、IP；设备名称优先来自 CMDB。
- [ ] 同一设备（IP/主机名相同）在一级列表只占一行；点击进入该设备备份详情。
- [ ] 设备备份详情内展示该设备下所有版本（最多 90 条）；支持删除、下载、选择任意两版对比。
- [ ] 业务展示用 ID 与作业关联（job_execution_id 或 job_id 写入并展示）。
- [ ] 同设备备份数超过 90 时采用**滚动覆盖**：先删该设备最旧一条再写入，不报错。
- [ ] 配置比较页对有差异的代码行使用高亮（如绿色底）展示。

| 审查建议中的点 | 本 PRD 对应 |
|----------------|-------------|
| 「与上一版对比」应为同设备版本对比 | F4.3、F5.4：设备详情内选任意两版对比 |

| version_no / created_by | 见原审查文档；本 PRD 聚焦展示与交互，ID 以作业执行 ID 为准 |
